vuepress的配置需要在docs目录下创建一个.vuepress目录，所有的配置文件放在.vuepress中

  • docs/.vuepress/public: 静态资源目录。

使用hexo搭建博客

参考：https://blog.csdn.net/weixin_45631738/article/details/104716374
前置条件和VuePress搭建博客流程相差不大，都是先安装NodeJs和git

  1. 先本地搭好环境，创建一个目录作为项目根目录，进行简单配置，快速启动

  2. 在github上创建一个私有仓库hexo-blog用来存放博客源码

  3. 在github上创建一个公开仓库用来存放博客编译后的静态资源

  4. 在本地对hexo-blog这个博客源码中的_config.yml进行配置

    在 _config.yml 中对 deploy 部分进行配置，指定要部署的仓库地址

  5. 配置好之后在本地博客源码目录进行资源生成和部署

  6. 部署时，会自动将编译后的静态资源push到配置好的远程仓库

  7. 在远程仓库的settings–Pages配置好即可，
    但实际发现资源加载错误，html没有样式，查阅官方文档后发现将仓库名称改为<username>.github.io即可

hexo配合GitHub Action自动部署博客

参考：https://cloud.tencent.com/developer/article/2201648

https://juejin.cn/post/6943895271751286821

https://sanonz.github.io/2020/deploy-a-hexo-blog-from-github-actions/

  1. 生成一个github的token

  2. 在私有仓库（就是博客源码对应的远程仓库）中将token添加到secrets中

  3. 创建一个action
    github 仓库页面选择actions – set up a workflow yourself

  4. 这样每次push到master分支，就会触发action种的jobs，就会自动将编译后的静态资源push到公共仓库
